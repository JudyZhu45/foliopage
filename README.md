# Foliopage

Local-first stock research in your browser — powered by Claude Code running on
your own subscription. Type a company name; get a fully self-contained HTML
research page with charts, fundamentals, news, and peer comparisons. Every
metric is a link that generates the next page on the fly.

No SaaS. No API keys. No streaming. Just complete pages, served locally.

---

## Demo

```
$ make dev
# Open http://localhost:8000
# Type: 贵州茅台
# Wait ~10 min (first run, cold cache)
# → Self-contained HTML: K-line chart, KPIs, financials, peer table, news
# Click any metric or peer → next page generates automatically
```

After the first run, the same query returns in under 100 ms (30-minute page cache).

## Quickstart

**Prerequisites:** Python 3.11+, [uv](https://docs.astral.sh/uv/), and an active
[Claude Code](https://claude.ai/code) subscription with the CLI installed
(`claude --version` should work).

```bash
git clone <repo-url> foliopage
cd foliopage

# Install dependencies and write .mcp.json for this machine
make install

# Start the server
make dev
```

Open [http://localhost:8000](http://localhost:8000) and type a stock name or code.

### Supported queries

| Query style | Example |
|-------------|---------|
| A-share name | `贵州茅台` |
| A-share code | `600519` |
| US ticker | `AAPL` |
| US company name | `Apple` |

## Architecture

```
Browser (index.html / report.html)
        │ HTTP POST /api/generate
        ▼
┌─────────────────────────────┐
│  Orchestrator (FastAPI)     │  orchestrator/server.py
│  • Page-level HTML cache    │
│  • Session management       │
│  • Concurrency semaphore    │
└───────────────┬─────────────┘
                │ subprocess  claude -p
                ▼
┌─────────────────────────────┐
│  Research Agent             │  claude CLI (your local subscription)
│  • Reads CLAUDE.md          │
│  • Calls MCP tools          │
│  • Writes HTML to disk      │
└──────┬────────┬─────────────┘
       │ stdio  │ stdio
   ┌───┘    ┌───┘
   ▼        ▼
stock_mcp  chart_mcp  news_mcp  cache_mcp
(akshare)  (matplotlib) (akshare) (SQLite)
```

Full design rationale: [`docs/architecture.md`](docs/architecture.md).

## Configuration

All settings are controlled by environment variables (or a `.env` file at repo root).

| Variable | Default | Description |
|----------|---------|-------------|
| `FOLIOPAGE_HOST` | `127.0.0.1` | Bind address |
| `FOLIOPAGE_PORT` | `8000` | Listen port |
| `FOLIOPAGE_MAX_CONCURRENT` | `3` | Max parallel agent runs |
| `FOLIOPAGE_AGENT_TIMEOUT` | `900` | Agent timeout in seconds |
| `FOLIOPAGE_PAGE_CACHE_TTL` | `1800` | Page cache TTL in seconds (0 = disable) |
| `FOLIOPAGE_PAGE_CACHE_ROOT` | `~/.foliopage/page_cache` | Page cache directory |
| `FOLIOPAGE_SESSION_ROOT` | `~/.foliopage/sessions` | Session workspace root |
| `FOLIOPAGE_CACHE_DB` | `~/.foliopage/cache.db` | SQLite cache path |
| `FOLIOPAGE_LOG_LEVEL` | `INFO` | Logging level |
| `FOLIOPAGE_CLAUDE_BIN` | `claude` | Path to the Claude CLI binary |

## Troubleshooting

**The page never appears / spinner runs forever**

- Check the terminal where `make dev` is running for errors.
- The agent transcript is at `~/.foliopage/sessions/<sess_id>/logs/transcript.jsonl`.
- First-run generation takes 8–12 minutes — this is normal.

**`claude: command not found`**

Install the Claude CLI and ensure it is on your `PATH`:
```bash
which claude    # should print a path
claude --version
```

**`Error: virtual environment not found`**

Run `make install` before `make dev`.

**`make test` fails with import errors**

Run `uv sync --all-extras` to install dev dependencies, then retry.

**Data shows "数据暂不可用"**

The upstream data source (akshare) returned an error. This is usually transient —
retry the query. If it persists for a specific stock, open an issue with the stock
code and the error from the transcript.
